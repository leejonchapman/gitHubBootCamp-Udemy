# A guide to reading Diffs

```diff
diff --git a/rainbow.txt b/rainbow.txt
index 72d1d5a..f2c8117 100644
--- a/rainbow.txt
+++ b/rainbow.txt
@@ -3,4 +3,5 @@ orange
 yellow
 green
 blue
-purple
+indigo
+violet
```

Below shows the two files that are being compared. It will declare one as A and the other as B.

```bash
diff --git a/rainbow.txt b/rainbow.txt
```

This shows the file metadata - it doesnt matter.

```bash
index 72d1d5a..f2c8117 100644
```

Markers are next - changes in file A will be highlighted with a -- sign

```bash
--- a/rainbow.txt
+++ b/rainbow.txt
```

This is called a Chunk header.

From file a (---), 4 lines are extracted starting from line 3.  
From file b (+++), 5 lines are extracted starting from line 3

```bash
@@ -3,4 +3,5 @@ orange
```

A Chunk - before and after a code change. If you changed 1000 lines of code, the diff wouldnt show you everything.

```bash
@@ -3,4 +3,5 @@ orange
 yellow
 green
 blue
-purple
+indigo
+violet
```

This would look like:

|Line|file a|file b|
|---|---|---|
|1|red|red|
|2|orange|orange|
|3|yellow|yellow|
|4|green|green|
|5|blue|blue|
|6|purple|indigo|
|7||violet|
