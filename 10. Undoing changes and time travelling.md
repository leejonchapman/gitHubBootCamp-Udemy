# Checking out old commits

```git
git checkout 
```

This is the swiss army knife of commands and can be used to do a lot of things such as:
- Switch to new branches
- Restore files
- Undo history

```git
git checkout d8ehdh2
```
This can be used to view a previous commit and you just need the first seven characters of the commit.

## Detached HEAD

**A detached HEAD state occurs when Git checks out a specific commit instead of a branch.** Normally HEAD points to a local branch reference. When you check out a commit directly, HEAD points directly to that unique commit id instead.

### What changes

- **Commit tracking** You can still make new commits. However no branch tracks them.
    
- **Moving away** If you switch to another branch your new commits become orphaned. Git will eventually delete them during garbage collection.
    

### Comparisons

|State|What HEAD points to|What happens when you commit|
|---|---|---|
|**Normal**|A branch pointer|The branch pointer moves forward with the new commit|
|**Detached**|A specific commit hash|HEAD moves forward but no branch updates|

### Questions you might have missed

**How do I fix a detached HEAD?** Switch back to your main branch using `git switch main`.

**How do I save work created in this state?** Create a new branch right where you are using `git switch c new branch name`.

**Is a detached HEAD dangerous?** No. You will not lose any existing code. Just remember to create a branch if you want to save new changes.
