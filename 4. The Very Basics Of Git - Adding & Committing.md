# What is a Git repository?
A Git repository manages and tracks changes to files in a folder.

Anytime you create a new project you will need to create a new Git repository and its full history will be tracked.

# Git Init and Git Status

Gives information about the current state of the repository and its contents.

# Mysterious .git folder
The `.git` folder is a hidden directory that stores your project's entire version history, configuration settings, and commit data.

**Should I ever modify files inside the `.git` folder manually?** Generally no, as altering these files directly can corrupt your repository database.

**Is the `.git` folder included when I push to GitHub?** No, GitHub receives the data inside it, but the actual folder stays local to your machine.

**How do I view this folder if it is hidden?** Run `ls -a` in your terminal or enable hidden items in your file manager

# A common early mistake
Do not init a new repository inside an existing repository, you can always check by running a **git status** command.

If you do this you can remove the **.git folder**.

# Committing the work flow
A **commit** is a saved snapshot of your project at a specific point in time. Think of it as a permanent save point in a video game. If I mess up later, I can reload this exact state.

A commit is **per repository**, you can group multiple files that have been edited together into a single commit.

The process is:
1. Work on the files you want to commit
2. **Add changes**, group specific files together in preparation for a commit
3. Commit the changes.
# Staging changes with Git Add
Moves your modified files from your working directory into the Staging Area.

Flow of a commit:
Working Directory(CRUD files) > Staging Area (Bundle in to a box) > Repository (Push box to repo)

It tells Git exactly which changes you want to include in your next repository commit. Think of it like packing a box, git add puts the items inside the box, and git commit tapes it shut and labels it.

Remember the **.git folder**? When you add a commit the data is stored in that folder.

**How do I check what is currently in my staging area?** Run `git status` to see files ready to be committed in green and unstaged files in red.

**How do I unstage a file if I added it by mistake?** Run `git restore --staged <file>` to pull it out of the staging area without losing your code changes.

**Does `git add` save my changes to the repository history?** No, it only prepares them. Your changes are not permanently saved until you run `git commit`.
